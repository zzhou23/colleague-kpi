## 一、指标体系升级

### 1.1 引入"AI 代码提交占比"指标（优先级：高）

**学习对象：** Cursor Analytics 的 "AI Share of Committed Code"

Cursor 在设备端对比 AI 建议签名与 Git diff，计算实际提交代码中 AI 生成的比例。业界基准：22-27%。

**实施思路：**
- Agent 采集时额外分析 Git 历史，对比 session 中 AI 生成的代码片段与实际 commit diff
- 新增评分维度：`ai_code_commit_ratio`（AI 代码产出转化率）
- 这比"使用时长"更能反映 AI 的实际价值

### 1.2 增加"对冲指标"防止刷量（优先级：高）

**学习对象：** DX Core 4 框架的对冲性指标设计

每个速度指标必须配对一个质量指标：
- 对话轮数多 → 但会话放弃率是否也高？
- Commit 产出多 → 但 CI 失败率是否也高？
- 活跃天数多 → 但有效会话占比如何？

**实施思路：**
- 在评分引擎中增加"质量系数"，当质量指标低于阈值时对速度指标打折
- 例：`effective_score = raw_activity_score × quality_coefficient`

### 1.3 新增"技能成长"维度（优先级：中）

**学习对象：** arXiv 论文 "Beyond the Commit" 提出的 6 维模型

论文警示：初级开发者过度依赖 AI 可能阻碍技能发展。

**新维度建议：**
- `ai_feature_diversity` — AI 功能使用多样性（补全/Chat/Agent/审查/规划），不只停留在聊天
- `prompt_evolution` — 提问质量随时间是否在提升（对话轮数趋势下降 = 提问越来越精准）
- `independence_trend` — 同类任务的 AI 依赖度是否逐月下降（学会了就不该再问）

### 1.4 新增"审查效率"维度（优先级：中）

**学习对象：** Faros AI 发现 PR 审查时间 +91%、PR 体积 +154%

AI 加速了代码生成，但审查成为新瓶颈。

**实施思路：**
- 如果能接入 Git 平台数据：跟踪 PR 审查周期、PR 体积变化
- 评估员工是否利用 AI 辅助做 Code Review（而非只用来写代码）

## 二、多 AI 工具支持（Phase 8）

### 2.1 GitHub Copilot 数据采集（优先级：高）

**学习对象：** GitHub Copilot Metrics API（2026年2月已 GA）

GitHub 官方提供组织级/用户级指标 API：
- DAU/WAU、代码补全接受率、AI 建议行数
- PR 生命周期指标、按模型/编辑器/语言分布
- 支持 NDJSON 导出

**实施思路：**
- 新增 `copilot_parser` 模块，通过 GitHub API 拉取 Copilot 指标
- 或在 Agent 端采集本地 Copilot 日志
- 参考开源项目：[`microsoft/copilot-metrics-dashboard`](https://github.com/microsoft/copilot-metrics-dashboard)

### 2.2 Cursor 数据采集（优先级：中）

**学习对象：** Cursor Analytics API

Cursor 企业版提供 Analytics API，包括：
- AI Share of Committed Code（最有价值的 ROI 指标）
- Agent Edits / Tab Completions 接受率
- 按功能的采用分布

**实施思路：**
- 新增 `cursor_parser` 模块
- 如果有企业版 API 权限可直接调用；否则采集本地日志

### 2.3 统一评分模型（优先级：中）

不同 AI 工具的原始指标不同，但评分维度应统一：
- 将 Copilot 的"补全接受率" 映射到 quality 类别
- 将 Cursor 的"AI 代码提交占比" 映射到 efficiency 类别
- 保持 5 大类 + 等级体系不变，只是数据来源多样化

## 三、组织级视角增强

### 3.1 团队维度分析（优先级：中）

**学习对象：** DORA 2025 的 7 种团队原型

当前系统只有个人评分。增加团队/部门聚合视图：
- 部门平均分对比、部门内分布
- 识别"AI 先进团队" vs "需要帮助的团队"
- 团队 AI 采用率热力图（日期 × 员工矩阵）

### 3.2 ROI 计算模块（优先级：低）

**学习对象：** DX 的三维 ROI 模型（利用率 × 影响力 × 成本）

**公式：**
```
净 ROI = (AI 节省时间 × 人均时薪) - AI 工具总支出
```

业界基准：每周人均节省 3.6 小时（DX Q4 数据）。

**实施思路：**
- Settings 页增加"人均时薪"和"AI 工具月费"配置
- Dashboard 增加 ROI 卡片

### 3.3 趋势预警系统（优先级：低）

- AI 使用量突然下降 → 自动告警（员工可能弃用）
- 质量指标持续走低 → 建议介入（可能过度依赖 AI）
- 新人入职 AI 熟练度曲线追踪（业界基准：入职时间从 91 天缩至 49 天）

## 四、数据质量与隐私

### 4.1 经验采样法（ESM）补充主观数据（优先级：低）

**学习对象：** DX Core 4 的三种数据收集法

当前系统只有系统指标（工具遥测），缺少主观体验数据。

METR 研究发现：开发者自认为用 AI 快了 20%，实际慢了 19%。纯客观数据也有盲区。

**实施思路：**
- 可选功能：定期向员工推送简短问卷（"这周 AI 帮你节省了多少时间？"）
- 将主观反馈作为评分的辅助参考（不影响硬指标）

### 4.2 隐私保护升级（优先级：中）

**学习对象：** Cursor 的设备端检测机制

Cursor 的 AI 代码占比计算完全在本地完成，源码不离开设备。

**实施思路：**
- Agent 端做更多预处理和聚合，只上传统计指标而非原始数据
- 增加"数据最小化"选项：企业可选择只采集聚合指标
